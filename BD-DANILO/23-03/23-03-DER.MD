# 📚 Modelagem DER (Resumo)

## 🔀 Especialização: Exclusiva vs Sobreposta

### 🔴 Exclusiva (Disjoint)
Uma entidade pode pertencer a **apenas uma subclasse**.

📌 **Exemplo:**
- Um registro **não pode ser** *Aluno* e *Professor* ao mesmo tempo

---

### 🟢 Sobreposta (Overlapping)
Uma entidade pode pertencer a **mais de uma subclasse simultaneamente**.

📌 **Exemplo:**
- Uma pessoa pode ser *Aluno* **e** *Professor*

---

## 🚗 Estrutura Generalizada

Representa uma **generalização/especialização**:

```text
Veículo
 ├── Carro
 ├── Moto
 └── Caminhão
```

<br>

♻️ Redundância de Atributos

Evite repetir atributos comuns em várias entidades.

📌 Solução:

Criar uma entidade genérica (superclasse)
🧱 Exemplo de DER: Entidade Fraca

📌 Cenário:

Pedido (1) contém Item_Pedido (N)

    erDiagram
        PEDIDO ||--o{ ITEM_PEDIDO : contem

        PEDIDO {
            int id_pedido PK
        }

        ITEM_PEDIDO {
            int id_item PK
            int id_pedido FK
        }

🧠 Observação:

- PEDIDO → entidade forte
- ITEM_PEDIDO → entidade fraca
- Chave composta: (id_pedido + id_item)

---

🔁 Exemplo de DER: Relacionamento Recursivo

📌 Descrição:

- Uma disciplina pode ter vários pré-requisitos
- Uma disciplina pode ser pré-requisito de várias outras

        erDiagram
            DISCIPLINA ||--o{ PRE_REQUISITO : tem
            DISCIPLINA ||--o{ PRE_REQUISITO : eh_requisito

            DISCIPLINA {
                int id_disciplina PK
                string nome
            }

            PRE_REQUISITO {
                int id_disciplina FK
                int id_pre_requisito FK
            }

🧠 Nota:

- Relacionamento recursivo ocorre quando uma entidade se relaciona com ela mesma
- Os papéis (roles) são essenciais para diferenciar

---

🧩 Exemplo de DER: Agregação

📌 Descrição:

- Funcionário trabalha em Projeto
- Esse relacionamento é supervisionado por um Departamento

🧠 Conceito:

- A agregação permite tratar um relacionamento como entidade

```text
        [FUNCIONARIO] --- trabalha_em --- [PROJETO]
                \________________________/
                    (agregação)
                            |
                    supervisionado_por
                            |
                    [DEPARTAMENTO]
```

🎯 Resumo Rápido
Conceito	Ideia principal
Entidade Fraca	Depende de outra para existir
Recursivo	Entidade se relaciona com ela mesma
Agregação	Relacionamento tratado como entidade
Exclusiva	Apenas uma subclasse
Sobreposta	Pode estar em várias subclasses
