# Interface vs Classe Abstrata

## Interface

A interface define um contrato que deve ser implementado pelas classes.

```csharp
public interface IPagamento {
    void processarPagamento(double valor);
}
```

------

## Características da Interface
- Define apenas o que deve ser feito
- Não possui construtor
- Permite herança múltipla
- Métodos são públicos por padrão
- Ideal para contratos e múltiplas implementações

-----

## Classe Abstrata

A classe abstrata pode implementar comportamentos comuns e também obrigar a implementação de métodos nas classes filhas.

-----

```csharp
public abstract class PagamentoBase {

    public void validarValor(double valor) {
        if (valor <= 0) {
            throw new Exception("Valor inválido");
        }
    }

    public abstract void processarPagamento(double valor);
}
```

-----

| Característica              | Interface                          | Classe Abstrata                          |
|-----------------------------|------------------------------------|------------------------------------------|
| Implementação de métodos    | Não                                | Sim (parcial)                            |
| Herança múltipla            | Sim                                | Não                                      |
| Construtor                  | Não                                | Sim                                      |
| Modificadores de acesso     | Apenas `public`                    | Diversos (`public`, `protected`, etc.)   |
| Uso principal               | Definir contrato                   | Fornecer comportamento base              |

-----

## Quando usar Interface

Use interface quando:

- precisar de múltiplas implementações
- quiser definir um contrato
- diferentes classes precisarem seguir o mesmo padrão

Exemplo:

```csharp
public class Pix implements IPagamento {

    @Override
    public void processarPagamento(double valor) {
        System.out.println("Pagamento via PIX");
    }
}
```

-----

## Quando usar Classe Abstrata

Use classe abstrata quando:

- existir comportamento comum para reutilizar
- quiser compartilhar código entre classes
- precisar forçar implementação de partes específicas

Exemplo:

```csharp
public class CartaoCredito extends PagamentoBase {

    @Override
    public void processarPagamento(double valor) {
        validarValor(valor);
        System.out.println("Pagamento no cartão");
    }
}
```

-----

### OBS metodo concreto é um metodo normal
