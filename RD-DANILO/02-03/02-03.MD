# Um pouco mais esplicativo o a entidade

Vamos tentar uma abordagem visual e física. Imagine que você é o dono de uma oficina mecânica.
O que é a Entidade na sua oficina?
A Entidade é o Carro.
Imagine que chega um Fusca Azul para consertar. Para a sua oficina não se perder, você cria uma Ficha de Papel para esse carro.
A "Entidade" é a Ficha: Ela representa o carro dentro do seu sistema.
O "ID" é a Placa: Mesmo que chegue outro Fusca Azul idêntico, a placa diz que aquele "objeto" é único.
Os "Atributos" são os dados na ficha: Modelo (Fusca), Cor (Azul), Ano (1970).
Por que não chamamos apenas de "carro"? Por que "Entidade"?
Porque na programação, a gente precisa separar o objeto físico da informação que o computador guarda.
O Carro (Mundo Real): É metal, pneu e óleo.
A Entidade (No Computador): É o conjunto de dados que "finge" ser o carro para o sistema poder fazer cálculos (ex: calcular o valor do conserto).
[!IMPORTANT]
A Regra de Ouro:
Se você consegue dizer "isso é um(a) [ALGO]" e esse algo tem características que você precisa salvar, esse algo é uma Entidade.
Vamos ver se você consegue identificar:
Imagine um sistema de uma Escola. O que seriam as entidades lá?
O Aluno é uma entidade? (Sim, ele tem nome, matrícula e precisa ser salvo).
O Professor é uma entidade? (Sim).
A Nota 10 é uma entidade? (Não! O 10 é só um número, um pedaço de informação que pertence à entidade Aluno).
Entidade é sempre o "Personagem Principal" daquela informação.
